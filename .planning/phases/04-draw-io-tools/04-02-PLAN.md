HEADER_H = 26    # swimlane startSize

**Private helpers:**

`_make_issue(issue, location, value=None, fix=None, severity="error") -> dict`
— mirrors tools/validation.py pattern exactly

`_class_height(n_attrs: int, n_methods: int) -> int`
— HEADER_H + max(n_attrs,1)*ROW_H + SEP_H + max(n_methods,1)*ROW_H

`_layout_for_canvas(n_vertices: int, edges: list[tuple[int,int]], canvas_w: int, canvas_h: int, margin: int = MARGIN) -> list[tuple[float,float]]`
— build igraph.Graph, call layout_sugiyama(), slice coords[:n_vertices], call fit_into((margin, margin, canvas_w-margin, canvas_h-margin)), return list of (x, y). Handle n_vertices==0 by returning []. Handle n_vertices==1 (no edges) by returning [(canvas_w//2, canvas_h//2)] without calling igraph.

`_attr_label(vis: str, scope: str, name: str, type_: str) -> str`
— same logic as in render_sample_xml: VIS_SYM = {public:"+", private:"-", protected:"#"}; underline if scope=="class"

`_method_label(vis: str, scope: str, name: str, params: list, return_type: str | None) -> str`
— format as "SYM name(p1: T1, p2: T2): ReturnType"; underline if scope=="class"

`_extract_drawio_ids(xml_path: Path) -> frozenset[str] | None`
— parse with lxml etree.parse(); collect all mxCell id attrs that contain ":"; return frozenset or None on XMLSyntaxError/missing file

`_compute_expected_class_ids(domain: str, cd: ClassDiagramFile) -> frozenset[str]`
— return frozenset of all IDs that render_to_drawio_class would generate: class_id, attribute_id (for attrs cell — use f"{class_id}:attrs" pattern), separator_id, f"{class_id}:methods", association_id for each assoc

`_structure_matches_class(domain_path: Path, domain: str, cd: ClassDiagramFile) -> bool`
— call _extract_drawio_ids; if None, return False; compare against _compute_expected_class_ids

`_build_class_diagram_xml(domain: str, cd: ClassDiagramFile) -> bytes`
— generate full mxfile XML using lxml etree (generalize render_sample_xml pattern):
  1. Build vertex list (class names) and edge list (associations as src_idx->tgt_idx)
  2. Compute canvas: canvas_w = max(1200, len(cd.classes)*280), canvas_h = max(800, 600)
  3. Call _layout_for_canvas to get (x,y) per class
  3.5. Apply greedy x-axis nudge pass to resolve horizontal overlaps (per user decision):
       Sort class indices by x position. For each adjacent pair (i, j) in sorted order,
       if x[j] - x[i] < CLASS_W, set x[j] = x[i] + CLASS_W. This is O(n log n) and
       ensures no two class boxes overlap horizontally. Use CLASS_W as the minimum gap.
  4. For each class in order: add swimlane cell (STYLE_CLASS, value="<<stereotype>>\nName"), attrs text cell, separator cell, methods text cell using absolute y positions
  5. For each association: add edge cell (STYLE_ASSOCIATION, source=class_id(domain, point_1), target=class_id(domain, point_2))
  6. Return etree.tostring(mxfile, encoding="unicode", xml_declaration=False).encode("utf-8")
  7. CRITICAL for idempotency: iterate classes in YAML list order (do not sort); set all attributes in SubElement() call; do NOT use pretty_print=True

**Public functions:**

`render_to_drawio_class(domain: str) -> list[dict]`
— 1) Resolve domain_path = MODEL_ROOT / domain; check exists
  2) Load class-diagram.yaml with yaml.safe_load + ClassDiagramFile.model_validate
  3) Call _structure_matches_class; if True, return [{"file": str(drawio_path), "status": "skipped"}]
  4) Call _build_class_diagram_xml
  5) Write to domain_path / "class-diagram.drawio"
  6) Return [{"file": str(drawio_path), "status": "written"}]
  All errors (missing domain, parse failure, Pydantic validation) go into issues list returned early

`render_to_drawio_state(domain: str, class_name: str) -> list[dict]`
— similar pattern for state-diagrams/<ClassName>.drawio:
  1) Load state-diagrams/<ClassName>.yaml + StateDiagramFile.model_validate
  2) _structure_matches_state (similar to class version, uses state_id, transition_id)
  3) _build_state_diagram_xml: initial pseudostate + states + transitions with Sugiyama layout
  4) Write to domain_path / "state-diagrams" / f"{class_name}.drawio" (mkdir parents=True)
  5) Return per-file result

For state diagram layout: vertices = [initial_pseudo] + [s.name for s in sd.states]; edges = [(0, initial_state_idx)] + [(from_state_idx, to_state_idx) for each transition]

Transition cell label format (three lines, matching render_sample_xml pattern):
```
{trans_cid}<br>{event}({param_sig})<br>[{guard}]
```
Only include guard line if guard is not None. Only include params if event has params (look up in sd.events).

`render_to_drawio(domain: str) -> list[dict]`
— 1) render_to_drawio_class(domain) -> accumulate results
  2) Load class-diagram.yaml; for each ClassDef with stereotype=="active": render_to_drawio_state(domain, cls.name) -> accumulate results
  3) Return combined result list (class diagram first, then state diagrams in class list order)
