## Decisions Made

- Used `_ikey_val` as local variable name inside the SenescentEntered block of `_process_event` to avoid shadowing the `ikey` variable already in scope for SenescentExited.
- `test_senescent_exited_on_wakeup` restructured to use `park` then `wake` transitions (Active->Idle->Active) rather than starting with initial-state senescence — `create_sync` via scenario does not dispatch a `__creation__` event through the scheduler, so `_process_creation` is never called and D-18 never fires. This is correct behavior (sync creation is a direct registry write, not a scheduler event).
- `test_initial_state_senescence` uses `SimulationContext.create_async()` directly so the `__creation__` event goes through the scheduler's `_process_creation` path.
- `test_determinism_identical_streams` relaxed per D-21: `EventCompleted.duration_ns` is real wall-clock nanoseconds and cannot be deterministic. Test now checks `target`, `name`, and `duration_ns > 0` separately.
