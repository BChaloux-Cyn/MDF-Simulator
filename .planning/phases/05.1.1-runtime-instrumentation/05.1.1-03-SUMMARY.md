## Decisions Made

- Used a for-loop over `scheduler.execute()` rather than `yield from` — only a for-loop can inspect each yielded value and conditionally emit additional steps after it. `yield from` delegates control entirely to the sub-generator with no interception point.
- Threshold read as `threshold = self.event_duration_warn_ns` inside the loop body (fresh read per iteration) — this gives D-08 mid-sim mutability at zero cost since attribute access is O(1).
- `test_long_event_warning_fires` constructs `SimulationContext` directly and sets `ctx.event_duration_warn_ns = 1` before calling `execute()` — using `run_simulation` would not allow attribute mutation between construction and execution.
- `threshold_ns=1` (1 nanosecond) is the test threshold — any real Python function call takes at least microseconds, so this is guaranteed to trigger without any `time.sleep()` needed.
