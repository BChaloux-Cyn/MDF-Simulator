### `engine/clock.py` (stateful service, request-response)

**Analog:** `engine/clock.py` itself — `advance()` method is the closest existing pattern.

**Current `__init__` and `advance`** (`engine/clock.py` lines 19–35):
```python
class SimulationClock:
    def __init__(self, speed_multiplier: float = 1.0):
        self._time_ms: float = 0.0
        self._speed_multiplier: float = speed_multiplier
        self._paused: bool = False

    def advance(self, delta_ms: float) -> None:
        if not self._paused:
            self._time_ms += delta_ms * self._speed_multiplier
```

**D-14 / D-15 / D-16 / D-18 target shape:**
```python
import time

class SimulationClock:
    def __init__(self, speed_multiplier: float = 1000.0, fast_forward: bool = False):
        self._time_ms: float = 0.0
        self._speed_multiplier: float = speed_multiplier
        self._fast_forward: bool = fast_forward
        self._paused: bool = False

    def advance_to(self, ms: float) -> None:
        """Jump simulation time to ms; sleep in real-time mode (D-15/D-18)."""
        if not self._fast_forward:
            delta = ms - self._time_ms
            if delta > 0:
                time.sleep(delta / (self._speed_multiplier * 1000.0))
        self._time_ms = max(self._time_ms, ms)
```
Critical: `self._time_ms = max(self._time_ms, ms)` — no `speed_multiplier` applied
to the time-set itself. The multiplier only scales the real-time sleep duration.

**`advance()` stays unchanged** — only used by internal clock ticks, not by delay
expiry advance. `advance_to()` is additive.

def _render_action_fn(fn_name: str, body_src: str, source_file: str, source_line: int) -> str:
    comment = f"# from {source_file}:{source_line}"
    if body_src and body_src.strip():
        lines = body_src.split("\n", 1)
        body = lines[1] if len(lines) > 1 else "pass"
    else:
        body = "pass"

    indented = textwrap.indent(body.strip() or "pass", "    ")
    return (

**D-02 replacement function** — rename or replace `_render_action_fn` with a new
emitter that produces `list[tuple[str, Callable]]`. Named inner functions per
statement are required (lambdas cannot contain Python statements):
```python
def _render_action_stmts(fn_name_prefix: str, body_src: str, source_file: str, source_line: int) -> str:
    """Emit list[tuple[str, Callable]] assignment for one action slot (D-02)."""
    comment = f"# from {source_file}:{source_line}"
    if not body_src or not body_src.strip():
        # D-06: empty body → single pass sentinel
        return (
            f"{comment}\n"
            f"def _stmt_0_{fn_name_prefix}(ctx, self_dict, params): pass\n"
            f'{fn_name_prefix} = [("pass", _stmt_0_{fn_name_prefix})]\n'
        )

    parser = _make_action_parser()
    tree = parser.parse(body_src)
    fn_defs: list[str] = []
    pairs: list[str] = []
    for i, stmt_tree in enumerate(tree.children):
        start = stmt_tree.meta.start_pos
        end = stmt_tree.meta.end_pos
        pycca_src = body_src[start:end].strip()
        transformer = ActionTransformer()
        py_body = transformer.transform(stmt_tree)
        indented = textwrap.indent(py_body.strip() or "pass", "    ")
        fn_label = f"_stmt_{i}_{fn_name_prefix}"
        fn_defs.append(
            f'def {fn_label}(ctx: "SimulationContext", self_dict: dict, params: dict) -> None:\n'
            f"{indented}\n"
        )
        pairs.append(f'    ({pycca_src!r}, {fn_label})')
    pairs_str = ",\n".join(pairs)
    return (
        f"{comment}\n"
        + "\n".join(fn_defs)
        + f"\n{fn_name_prefix} = [\n{pairs_str}\n]\n"
    )
```
Note: `_make_action_parser` already uses `propagate_positions=True`
(`compiler/transformer.py` line 845–846), so `stmt_tree.meta.start_pos` and
`stmt_tree.meta.end_pos` are character offsets into `body_src`. Verify this
assumption — see RESEARCH.md Assumption A2.

**Caller site in `generate_class_module`** (`compiler/codegen.py` lines 255–256):
```python
block = _render_action_fn(fn_name, transformed, source_file, 0)
```
Replace with:
```python
block = _render_action_stmts(fn_name, transformed, source_file, 0)
```
The `transformed` variable already holds the full body_src (after `transform_action`).
Under D-02, the per-statement parse happens inside `_render_action_stmts` using the
raw `action_src` (before `transform_action`). Adjust the call chain accordingly: pass
the raw `action_src` string to `_render_action_stmts` and remove the pre-call to
`transform_action`.

**TRANSITION_TABLE emission** (`compiler/codegen.py` lines 161–164): no structural

## Shared Patterns

### Frozen dataclass field addition
**Source:** `engine/event.py` (default-valued trailing fields) and `engine/microstep.py`
(Literal `type` field with `init=False`)
**Apply to:** `engine/event.py` (add `event_id`), `engine/microstep.py` (add base
`step_id`/`timestamp_ms` and `ActionExecuted` positional fields, propagate `event_id`
to four subclasses)

Pattern: new fields with defaults go after all required positional fields; `init=False`
fields always last. `kw_only=True` only on the base class fields so subclasses retain
positional construction.

### `dataclasses.replace()` for frozen mutation
**Source:** Python stdlib idiom; used nowhere yet in engine/ — but is the mandated
approach per RESEARCH.md.
**Apply to:** `engine/scheduler.py` (stamping all yielded steps; assigning `event_id`
at enqueue time)
```python
import dataclasses
stamped = dataclasses.replace(step, step_id=counter, timestamp_ms=ts)
event = dataclasses.replace(event, event_id=self._event_id_counter)
```

### Instance snapshot/diff for assignments_made
**Source:** No existing analog — new pattern introduced in D-05.
**Apply to:** `engine/scheduler.py` `_process_event` and `_process_creation`
per-statement loop.
```python
pre = dict(instance)
stmt_fn(ctx, instance, args)
assignments_made = {k: v for k, v in instance.items() if pre.get(k) != v}
```
Critical: snapshot BEFORE calling `stmt_fn`, diff AFTER.

### Soft-fail / warning pattern for compiler transform errors
**Source:** `compiler/codegen.py` lines 246–251
```python
warnings.warn(
    f"Failed to compile action for state {state_name}: {exc}",
    stacklevel=2,
)
transformed = ""
```
**Apply to:** Any new per-statement transform loop in `_render_action_stmts` — wrap
each `ActionTransformer().transform(stmt_tree)` call in the same try/warn pattern,
emitting a `pass` function for that statement on failure.
