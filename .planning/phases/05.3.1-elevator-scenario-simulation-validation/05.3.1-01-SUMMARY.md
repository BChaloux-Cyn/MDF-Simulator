# Phase 05.3.1 Plan 01: Compiler Pipeline Bug Fixes Summary

Five structural bugs in the compiler pipeline fixed so generated code correctly maps xUML entry-action semantics and the transition table preserves all guarded branches.

## Tasks Completed

### Task 1: Fix action-assignment direction, enum scoping, None keyword, guard sibling table format, and silent exception handling

All seven sub-fixes applied:

**Fix A: Transition table format** — `engine/manifest.py` type annotation and `compiler/manifest_builder.py` `_build_transition_table` changed from single `TransitionEntry` to `list[TransitionEntry]` per key. Uses `table.setdefault(key, []).append(entry)` to preserve all sibling branches.

**Fix B: Action-assignment direction** — `compiler/codegen.py` `generate_class_module` now maps `action_fn_map[(s,e)] = fn_name` when `entry["next_state"] == state_name` (incoming transitions), not when `s == state_name` (outgoing). This is the correct xUML entry-action semantics.

**Fix C: Enum scoping** — All types from `type_registry` are emitted in every class module, not just those referenced by attributes. Enum members may be used in action bodies.

**Fix D: None keyword conflict** — `_render_enum` escapes Python keywords: `None` → `None_`, `True` → `True_`, `False` → `False_`.

**Fix E: Silent exception swallowing** — Bare `except Exception: pass` replaced with `warnings.warn(f"Failed to compile action for state {state_name}: {exc}")`.

**Fix F: Codegen transition table rendering** — `_render_transition_table` and guard_fn_map loop updated to handle list-valued entries with index-qualified keys for guard siblings.

**Fix G: Scheduler guard chain** — `engine/scheduler.py` `_process_event` now iterates `entries` list, evaluating guards in order and selecting the first passing branch. Unguarded entries serve as fallback.

**Additional fixes found during execution:**

- `engine/bundle_loader.py` rebind loop updated for list format (Rule 1 — bug)
- `compiler/__init__.py` type_registry population fixed: was using `getattr(types_raw, "enums", [])` which always returned empty; now iterates `types_raw.types` correctly (Rule 1 — bug)
- `compiler/codegen.py` `_render_typedef` added xUML→Python primitive mapping to prevent `NameError` in generated modules using `Integer`, `Real`, etc. (Rule 1 — bug)
- Updated test fixtures in `test_engine.py`, `test_compiler_bundle.py`, `test_compiler_manifest.py`, `test_simulation_runner.py` to use list format
