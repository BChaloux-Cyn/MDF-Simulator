## What Was Built

### 5 Scenario YAML files

Each scenario uses trigger-based state-entry bypasses to avoid SimulationClock delay dependencies:

- **Scenario 1** (`elevator_scenario_1_door_cycle.scenario.yaml`): CarDoor + ShaftDoor full open/close cycle driven by `Open_cmd` to CarDoor. Triggers bypass `Done_opening`, `Close_cmd`, `Done_closing` delays.
- **Scenario 2** (`elevator_scenario_2_direct_travel.scenario.yaml`): Direct `Floor_assigned` to Elevator. Trigger injects `Floor_reached(3)` when Moving state entered. Full travel path: `Idle -> Departing -> Moving -> Floor_Updating -> Arriving -> Exchanging -> Idle`.
- **Scenario 3** (`elevator_scenario_3_button_travel.scenario.yaml`): `Activated` to FloorCallButton at floor 3. Lit entry action traverses R6->R3->R2->R1 to find Idle elevator, sends `Floor_assigned`. Same travel path as Scenario 2.
- **Scenario 4** (`elevator_scenario_4_full_e2e.scenario.yaml`): Button press + full travel + complete door cycle. Combines Scenarios 2 and 3.
- **Scenario 5** (`elevator_scenario_5_round_trip.scenario.yaml`): FloorCallButton at floor 3 pressed; elevator travels from floor 1. Includes floor 1 + floor 3 ShaftDoor instances.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] type_registry was never populated (types_raw.enums/typedefs attributes don't exist)**
- **Found during:** Task 2 debugging тАФ `Up`/`Down` bare name NameError in generated code
- **Issue:** `compiler/__init__.py` used `getattr(loaded.types_raw, "enums", [])` and `getattr(loaded.types_raw, "typedefs", [])`, but `TypesFile` has a `.types` list of `EnumType`/`ScalarType` objects (not `.enums`/`.typedefs` attributes). Result: type_registry always empty, no enums generated.
- **Fix:** Iterate `loaded.types_raw.types`; check `entry.base == "enum"` to distinguish enum vs scalar
- **Files modified:** `compiler/__init__.py`
- **Commit:** 44c9c2f

**2. [Rule 1 - Bug] `_render_enum` emitted `None = "None"` causing SyntaxError in generated modules**
- **Found during:** Task 2 тАФ scenario 1 (door cycle) broke after type_registry fix
- **Issue:** Direction enum has `None` as a member value. `class Direction(enum.Enum): None = "None"` is a SyntaxError тАФ `None` is a Python keyword.
- **Fix:** Skip Python keywords (`None`, `True`, `False`) both as enum member definitions and as module-level aliases
- **Files modified:** `compiler/codegen.py`
- **Commit:** 44c9c2f

**3. [Rule 2 - Missing functionality] Module-level enum member aliases needed for generated action code**
- **Found during:** Task 2 тАФ `NameError: name 'Up' is not defined` in `action_Departing_entry`
- **Issue:** Pycca action bodies reference enum member names as bare identifiers (e.g., `self.direction = Up`). Generated code uses those identifiers directly but they aren't in scope.
- **Fix:** `_render_enum` emits `Up = Direction.Up` etc. as module-level aliases after the class definition
- **Files modified:** `compiler/codegen.py`
- **Commit:** 44c9c2f

**4. [Rule 1 - Bug] State name comparisons in YAML used bare identifiers (Lit, Idle) that fail at runtime**
- **Found during:** Task 2 тАФ would have produced `NameError: name 'Lit' is not defined` at runtime
- **Issue:** `btn.curr_state == Lit` and `elev.curr_state == Idle` тАФ state names are stored as strings like `"Lit"` in `curr_state`; comparing against bare Python identifier `Lit` (undefined) always fails
- **Fix:** Changed YAML action bodies to use string literals: `== "Lit"`, `== "Idle"`
- **Files modified:** `Elevator.yaml`, `FloorCallButton.yaml`
- **Commit:** 44c9c2f

**5. [Rule 1 - Bug] `final_states` incorrectly included all states without entry actions**
- **Found during:** Task 2 тАФ `double_deletion` error in trace for Elevator after reaching Idle
- **Issue:** `manifest_builder.py` computed final states as "states with no entry action OR containing 'delete'". `Idle` and `Moving` have no entry actions but are normal resting/active states, not lifecycle-final states. The engine auto-deletes instances reaching final states, causing double-deletion when Elevator reached Idle.
- **Fix:** Final states = states with NO outgoing transitions in the transition table (true xUML final state semantics)
- **Files modified:** `compiler/manifest_builder.py`
- **Commit:** 44c9c2f
