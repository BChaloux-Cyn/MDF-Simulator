## Fixed Issues

### WR-01: `tmpdir` leaked on `BundleVersionError` and `BundleCorruptError`

**Files modified:** `engine/bundle_loader.py`
**Commit:** 56273c7
**Applied fix:** Wrapped the entire `load_bundle` body (after `tmpdir` creation) in a
`try/except Exception` block. The `except` branch calls
`shutil.rmtree(tmpdir, ignore_errors=True)` then re-raises, ensuring the temp directory
is cleaned up on every error path. Added `import shutil` to the module imports.

### WR-02: Path traversal guard bypassable on Windows due to case-sensitive `startswith`

**Files modified:** `engine/bundle_loader.py`
**Commit:** 56273c7
**Applied fix:** Changed the traversal guard to lower-case both the `tmpdir` prefix and
each resolved entry path before comparison, and appended `os.sep` to the prefix string
so a path that merely shares a prefix string with the temp dir cannot pass. Added
`import os` to the module imports. Both WR-01 and WR-02 were applied to the same file
and committed together.

### WR-03: Transform failures silently swallowed â€” malformed action bodies produce silent no-ops

**Files modified:** `compiler/codegen.py`
**Commit:** ce26289
**Applied fix:** Documented the intentional soft-fail design decision in both the action
and guard `except` blocks with inline comments explaining: (a) this is deliberate policy,
(b) the `UserWarning` is the notification mechanism, (c) the upgrade path to a hard error
is to raise `CompilationFailed` and accumulate via the caller. Changing behavior to
hard-fail would break the existing `test_compile_failure_emits_warning` test, which
explicitly verifies the soft-fail contract.

### WR-04: `test_run_scenario_yields_micro_steps_and_resolves_aliases` â€” transition never confirmed to fire

**Commit:** 69c0e0d

missing `TransitionFired` assertion: imports `TransitionFired` from `engine.microstep`,
filters `steps` for instances, asserts exactly one fired, and asserts
`fired[0].to_state == "Departing"`. Test confirmed passing after the change.
