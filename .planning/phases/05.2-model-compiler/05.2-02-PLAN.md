<objective>
Build the Lark Transformer that converts parsed pycca action bodies and guards into Python source strings. This is the codegen core: every other plan in the phase consumes its output. Per D-01, transformer returns strings (no `ast` module, no `exec`).

Purpose: Translate Tier 3 SYNTAX.md surface (D-08) for action bodies and guards while threading source location info for D-05 comments and accumulating errors per D-09.
Output: compiler/ package skeleton, error accumulator, ActionTransformer + GuardTransformer.
</objective>

> **D-10 Override (guard signature):** This plan emits 2-arg guard signature `def guard_<name>(self_dict: dict, params: dict) -> bool` per **scheduler ground truth**. CONTEXT.md D-10 specified 3-arg `(ctx, self, params)` but `engine/scheduler.py` calls `guard_fn(self_dict, params)` with exactly two positional arguments — 3-arg guards would fail at runtime. Action functions remain 3-arg `(ctx, self_dict, params)` per D-10. This override is recorded in 05.2-RESEARCH.md "Open Questions (RESOLVED)" §2 and is amended into CONTEXT.md D-10 by Task 1 below.

    Class `ActionTransformer(lark.Transformer)`:
    - Method per grammar rule used by the elevator model: `name`, `dotted_name`, `assignment`, `compare_expr`, `add_expr`, `mul_expr`, `or_expr`, `and_expr`, `not_expr`, `if_stmt`, `for_each_stmt`, `generate_stmt` (immediate + delayed), `cancel_stmt`, `create_stmt`, `delete_stmt`, `relate_stmt`, `unrelate_stmt`, `select_related_stmt`, `select_expr`, `bridge_call`, `method_call_stmt`, `lambda_expr`, `capture_list`, `capture_item`, `traversal_chain`, `block`, `start`.
