## Design revision (mid-plan)

The original plan used `yaml_schema` objects as compiler input. This was revised to use `CanonicalClassDiagram` / `CanonicalStateDiagram` (the same canonical JSON representation used for YAML‚ÜîDraw.io comparison). This keeps the seam clean: the compiler never touches `yaml_schema` directly.
