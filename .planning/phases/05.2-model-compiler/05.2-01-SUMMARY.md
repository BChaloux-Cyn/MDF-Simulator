## What Was Built

### Grammar Fixes (pycca/grammar.py)

Four grammar gaps identified in 05.2-RESEARCH.md were closed:

**Gap 1 — Lambda generic return type (`-> Set<T>`):**
Introduced `LAMBDA_RETURN_TYPE.3` terminal (priority 3, regex `NAME(?:<NAME,...>)?`) used directly in `lambda_expr`. The `type_expr` rule was left with a `GENERIC_TYPE.2` terminal for other positions, but the lambda return position uses its own terminal. Root cause: xearley's dynamic scanner tries `NAME` before `GENERIC_TYPE` at the same position because they share the same prefix (`[a-zA-Z_]`). An atomic terminal with higher priority than `NAME` resolves this.

**Gap 2 — Capture list dotted names (`[self.attr]`):**
Introduced `capture_item: NAME "." NAME | NAME` sub-rule. Changed `capture_list` to use `capture_item` instead of bare `NAME`. Supports `[self.current_floor]` and `[my_var, self.attr]` forms.

**Gap 3 — Multi-hop select related (`self->R1->R2`):**
Changed `select_related_stmt` to reuse existing `traversal_chain` rule. Single-hop (`self->R7`) remains a degenerate traversal_chain. Consistent with `select_expr` which already used `traversal_chain`.

**Gap 4 — Named bridge call arguments (`Domain::Op[key: expr]`):**
Added `named_arg_list: NAME ":" expr ("," NAME ":" expr)*` rule. Updated `bridge_call` to three alternatives: named_arg_list, arglist, or empty `[]`. Backward compatible — positional and empty calls unchanged.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] LAMBDA_RETURN_TYPE terminal required instead of type_expr in lambda return position**
- **Found during:** Task 1, Gap 1 fix
- **Issue:** `type_expr` with `GENERIC_TYPE` terminal failed in xearley because the dynamic scanner matched `NAME` (short prefix) before trying `GENERIC_TYPE` (same prefix + `<T>` suffix). Terminal priority `.2` on `GENERIC_TYPE` was insufficient — xearley's scan loop does not retry higher-priority terminals when a lower-priority one already matched.
- **Fix:** Introduced separate `LAMBDA_RETURN_TYPE.3` terminal (priority 3 > NAME priority 0) used exclusively in the `lambda_expr` rule. The terminal regex `[a-zA-Z_][a-zA-Z0-9_]*(?:<[a-zA-Z_][a-zA-Z0-9_,\s]*>)?` matches both `bool` and `Set<T>` atomically.
- **Files modified:** pycca/grammar.py
- **Commit:** f048b2d

**2. [Rule 1 - Bug] Tests must run from worktree directory (sys.path)**
- **Found during:** Task 1 verification
- **Issue:** Initial test run used `/c/.../mdf-sim` as CWD, importing the main repo's `pycca/grammar.py` (unpatched). The worktree's patched grammar was invisible.
- **Fix:** All test runs use `.venv/Scripts/python.exe` with CWD set to the worktree root. The worktree directory is on `sys.path` by default when pytest is invoked from there.
- **Files modified:** None (process fix)
