  <action>
    Implement `compiler/packager.py` per Research Pattern 4:

    - `compute_model_hash(generated_files: dict[str, str], manifest: dict) -> str` — sha256 over `json.dumps(sorted(...), ensure_ascii=True)` of (sorted file contents) + (sorted manifest). No timestamps anywhere.

    - `write_bundle(domain_name, generated_files, manifest, output_dir, engine_version, pycca_version) -> Path`:
      - Build `bundle_meta = {"engine_version": ..., "pycca_version": ..., "model_hash": ...}`
      - Open `ZipFile(<output_dir>/<domain>.mdfbundle, "w", ZIP_DEFLATED)`
      - Write entries in this order: bundle.json, manifest.json, generated/__init__.py, then `for class_name in sorted(generated_files): zf.writestr(f"generated/{class_name}.py", ...)`
      - Strip callables from manifest before serializing (callables are rebound by Phase 5.3 loader per D-02)
      - Return the bundle path

    Update `compiler/__init__.py` `compile_model(model_root, output_dir) -> Path`:
      1. `loaded = load_model(model_root)` (raises CompilationFailed on errors)
      2. `parser = Lark(PYCCA_GRAMMAR, ..., propagate_positions=True)`
      3. `manifest = build_domain_manifest(loaded, parser)`
      4. `generated_files = {cls.name: format_source(generate_class_module(cls, type_registry, parser), filename=f"{cls.name}.py") for cls in manifest.classes}`
      5. `init_src = format_source(generate_init_module(manifest), "generated/__init__.py")`
      6. `generated_files["__init__"] = init_src`
      7. Strip callables from manifest before passing to `write_bundle`
      8. `return write_bundle(domain_name, generated_files, manifest_serializable, output_dir, engine_version, pycca_version)`

    **Version constants (committed strategy — no ambiguity):**
    - Define `COMPILER_VERSION = "0.1.0"` as a module-level constant in `compiler/__init__.py`. This is the single source of truth for the compiler version.
    - `bundle.json` includes `"compiler_version": COMPILER_VERSION`.
    - Also include `"engine_version_required": COMPILER_VERSION` so the Phase 5.3 loader can refuse incompatible bundles by string compare. The compiler does NOT import from `engine/*` at runtime (preserves D-11 boundary) — the version pin is asserted by the loader, not the compiler.
    - `pycca_version`: hardcode `PYCCA_VERSION = "0.1.0"` as a constant in `compiler/__init__.py` next to `COMPILER_VERSION`. **Do NOT** use `importlib.metadata` and **do NOT** use try/except imports of `engine` or `pycca` — both were rejected as ambiguous and as a D-11 boundary risk.

    Drive each behavior bullet with a unit test before implementing.
