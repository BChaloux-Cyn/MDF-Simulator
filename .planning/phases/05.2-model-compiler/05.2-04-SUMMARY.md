## Design decisions applied

- **D-01**: String templates + black; no ast module; no exec
- **D-02**: `bundle.json` contains `compiler_version`, `engine_version`, `pycca_version`, `model_hash` — no timestamps
- **D-05**: Every emitted function preceded by `# from <model_file>:<line>` comment
- **D-06**: Enum → `enum.Enum` subclass; typedef → `typing.NewType`
- **D-07**: All dicts/iterations sorted; ZipInfo with fixed date_time for byte-identical zips
- **D-10**: Action sig `(ctx, self_dict, params) -> None`; guard sig `(self_dict, params) -> bool`
- **D-11**: `engine/*` only under `TYPE_CHECKING` in generated code; never in compiler itself
- **D-12**: Bundle layout: `bundle.json`, `manifest.json`, `generated/__init__.py`, `generated/<Class>.py`

## Key implementation notes

- **Tuple key serialization**: `transition_table` has `(state, event)` tuple keys; `_strip_callables` converts them to `"state::event"` strings for JSON. Phase 5.3 loader reverses this.
- **Determinism**: Fixed `_ZIP_DATE = (2020, 1, 1, 0, 0, 0)` in every `ZipInfo` entry eliminates the only non-deterministic field in zip byte streams.
- **Version constants**: `COMPILER_VERSION = "0.1.0"` and `PYCCA_VERSION = "0.1.0"` hardcoded in `compiler/__init__.py` — no `importlib.metadata`, no `engine/` import (D-11 boundary clean).

## compile_model pipeline

```
load_model(model_root)
    → build_domain_manifest(loaded, STATEMENT_PARSER)
    → generate_class_module(cls_manifest, type_registry, parser) per class
    → format_source(raw_src) via black
    → write_bundle(domain_name, files, manifest, output_dir, ...)
    → <domain>.mdfbundle
```
