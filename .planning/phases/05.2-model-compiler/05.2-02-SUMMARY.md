# Phase 5.2 Plan 02: Compiler Transformer + Error Accumulator Summary

**One-liner:** Lark Transformer codegen core — pycca action bodies and guard expressions compile to Python source strings with `# from file:line` audit trail and full error accumulation.

## What Was Built

### Task 1: compiler/ scaffold + CompileError accumulator

- `compiler/__init__.py` — public API package with `compile_model` placeholder (NotImplementedError, filled by Plan 04). Re-exports `CompileError`, `ErrorAccumulator`. D-11 no-engine-import enforcement comment at top.
- `compiler/error.py` — `CompileError` frozen dataclass formatting as `file:line: message` per D-09. `ErrorAccumulator` with `add/extend/raise_if_any`. `CompilationFailed` exception bundles all messages in a single raise.

### Task 2: ActionTransformer + GuardTransformer

- `compiler/transformer.py` — 700+ line Lark Transformer covering every pycca construct used by the elevator model:
  - Assignment rules: `self.attr = expr` → `self_dict["attr"] = expr`; `rcvd_evt.field` → `params["field"]`
  - Control flow: `if_stmt` with else_body disambiguation; `for_each_stmt`; `return_stmt`
  - Generate: all 6 grammar alternatives; `self` target → `self_dict["__instance_key__"]`; delay_ms conversion
  - Select: `select_stmt`, `select_related_stmt`, `select_expr` — all using CARDINALITY token
  - Bridge: `bridge_call` with `named_arg_list` (named args) → `ctx.bridge("Domain", "Op", {...})`
  - Traversal: `traversal_chain` → `ctx.traverse(source, ["R1", "R2"])`
  - Lambda, capture_list, method_call, create, delete, relate, unrelate, cancel
  - `GuardTransformer` inherits ActionTransformer; blocks statement-form rules
  - `transform_action(source, file, offset) -> str` and `transform_guard(source, file, line) -> str` top-level helpers with D-05 source line comments
  - `__default__` raises NotImplementedError naming the rule (T-05.2-05 mitigation)
  - Tier 3 stubs (while, switch, set ops) raise NotImplementedError (D-08 fidelity)

- **Commit:** f8863c3

**2. [Rule 1 - Bug] Earley parser flattens if-body and else-body into one list**
- **Found during:** Task 2 — `test_if_else` failed: else body was appended to if body
- **Issue:** Grammar rule `if_stmt: ... "{" statement* "}" ... ("else" "{" statement* "}")?` creates structural ambiguity. Earley returns `[cond, stmt1, stmt2]` with no way to split if-body from else-body.
- **Fix:** Added `else_body: statement+` named rule; updated `if_stmt` to use it; added `else_body` transformer method using `__else__:` sentinel prefix.
- **Files modified:** pycca/grammar.py, compiler/transformer.py
- **Commit:** 92c84ff (part of)

**3. [Rule 1 - Bug] `("any"|"many")` anonymous literals dropped from parse tree**
- **Found during:** Task 2 — `test_select_any_related` failed: `select_related_stmt` received only 2 children (var, chain), no cardinality
- **Issue:** Lark drops anonymous string literal alternatives. `("any"|"many")` never appeared as a child Token, making cardinality inaccessible to the transformer.
- **Fix:** Added `CARDINALITY: "any" | "many"` terminal; updated `select_stmt`, `select_related_stmt`, `select_expr` to use it.
- **Files modified:** pycca/grammar.py
- **Commit:** 92c84ff (part of)

**4. [Rule 1 - Bug] Token children arrive as strings, not Token objects, in Transformer**
- **Found during:** Task 2 — `test_generate_to_self` failed: `target=self` instead of `target=self_dict["__instance_key__"]`
- **Issue:** `__default_token__` converts all Token children to `str` before rule methods run. `isinstance(child, Token)` checks in `generate_stmt` always returned False.
- **Fix:** Changed `generate_stmt` to use string identity checks (`_is_param_dict`, `_normalize_target`) instead of Token isinstance checks.
- **Files modified:** compiler/transformer.py
- **Commit:** 92c84ff (part of)

**5. [Rule 1 - Bug] Tier 3 test expected CompilationFailed/NotImplementedError but got LarkError**
- **Found during:** Task 2 — `test_while_stmt_raises_compile_error` failed with `UnexpectedCharacters`
- **Issue:** `while` is not in the grammar at all, so the parse fails before the transformer runs. The D-08 intent (reject with clear error) is satisfied — just at parse time.
- **Fix:** Updated test to accept `lark.LarkError` in addition to `CompilationFailed` and `NotImplementedError`.
- **Files modified:** tests/test_compiler_transformer.py
- **Commit:** 92c84ff (part of)
