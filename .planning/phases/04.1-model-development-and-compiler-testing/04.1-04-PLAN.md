Current grammar.py PYCCA_GRAMMAR string (complete, confirmed by inspection):

    start: statement+

    statement: assignment
             | generate_stmt
             | bridge_call
             | create_stmt
             | delete_stmt
             | select_stmt
             | if_stmt

    assignment: "self" "." NAME "=" expr ";"
    generate_stmt: "generate" NAME "to" ("SELF" | "CLASS") ";"
    bridge_call: NAME "::" NAME "[" arglist? "]" ";"
    create_stmt: "create" "object" "of" NAME ";"
    delete_stmt: "delete" "object" "of" NAME "where" expr ";"
    select_stmt: "select" ("any"|"many") NAME "from" "instances" "of" NAME ("where" expr)? ";"
    if_stmt: "if" expr ";" statement* "end" "if" ";"

    ?expr: simple_compare | and_expr | or_expr | cardinality_expr | atom
    simple_compare: atom OP atom
    and_expr: expr "and" expr
    or_expr: expr "or" expr
    cardinality_expr: "cardinality" NAME
    atom: NUMBER -> number | ESCAPED_STRING -> string | NAME -> name | "(" expr ")"
    arglist: expr ("," expr)*
    OP: /[less-than greater-than]=?|==|!=/
    NUMBER: /[0-9]+(\.[0-9]+)?/
    NAME: /[a-zA-Z_][a-zA-Z0-9_]*/
    %import common.ESCAPED_STRING
    %ignore /\s+/
    %ignore /\/\/[^\n]*/

Nine gaps to close (from RESEARCH.md Grammar Extension Guide):
  1. else/elif in if_stmt
  2. select any var related by self->R11 (select_related_stmt)
  3. generate Event to var (variable target, not just SELF/CLASS)
  4. generate Event(param: val) to var (params)
  5. var.attribute reads (non-self dotted access in expr)
  6. relate/unrelate statements
  7. delete var (variable form)
  8. create var of Class(p: v) (create with params)
  9. self.attr assignment with var.attr on RHS (needed for var.attr reads)

GUARD_PARSER is start="expr" with Earley — more permissive; dot-access in expr is also needed.

The existing grammar uses LALR for STATEMENT_PARSER. LALR has strict ambiguity constraints.
Key known issue: if/else with arbitrary nesting can cause LALR shift-reduce conflicts.
Use Earley fallback (already in place in grammar.py) if LALR fails after extension.

Current grammar.py fallback pattern:

        STATEMENT_PARSER = Lark(PYCCA_GRAMMAR, start="start", parser="lalr")
    except Exception:
        STATEMENT_PARSER = Lark(PYCCA_GRAMMAR, start="start", parser="earley")
This fallback must be preserved.

Extend the PYCCA_GRAMMAR string in grammar.py by adding new rules. Apply changes in this
order, running pytest tests/test_pycca.py after each addition:

EXTENSION 1 — dot-access in atoms (for rcvd_evt.floor_num, req.destination_floor, self.attr):
Replace the atom rule to add a dotted-name variant:
    atom: NUMBER -> number
        | ESCAPED_STRING -> string
        | NAME "." NAME -> dotted_name
        | NAME -> name
        | "(" expr ")"
NOTE: dotted_name must appear BEFORE plain name or LALR may not see it. Test after adding.

EXTENSION 2 — generate_stmt: allow any NAME as target (variable or SELF/CLASS):
Replace: generate_stmt: "generate" NAME "to" ("SELF" | "CLASS") ";"
With:    generate_stmt: "generate" NAME "to" NAME ";"
         | "generate" NAME "(" param_list ")" "to" NAME ";"
Where:   param_list: NAME ":" expr ("," NAME ":" expr)*

EXTENSION 3 — select_related_stmt (relationship traversal select):
Add new rule:
    select_related_stmt: "select" ("any"|"many") NAME "related" "by" NAME "->" NAME ";"
Add select_related_stmt to the statement alternatives.

EXTENSION 4 — relate and unrelate statements:
Add:
    relate_stmt: "relate" NAME "to" NAME "across" NAME ";"
    unrelate_stmt: "unrelate" NAME "from" NAME "across" NAME ";"
Add both to statement alternatives.

EXTENSION 5 — delete variable form:
Replace: delete_stmt: "delete" "object" "of" NAME "where" expr ";"
With:    delete_stmt: "delete" "object" "of" NAME "where" expr ";"
               | "delete" NAME ";"
(Two alternatives for delete_stmt.)

EXTENSION 6 — create with params:
Replace: create_stmt: "create" "object" "of" NAME ";"
With:    create_stmt: "create" NAME "of" NAME ";"
               | "create" NAME "of" NAME "(" param_list ")" ";"
(Old "create object of" syntax becomes invalid — "object" is now the variable name.
This aligns with pycca/SYNTAX.md section 1.3.)

EXTENSION 7 — if_stmt with else and elif:
Replace: if_stmt: "if" expr ";" statement* "end" "if" ";"
With brace syntax per SYNTAX.md section 1.8:
    if_stmt: "if" "(" expr ")" "{" statement* "}" ("else" "{" statement* "}")?

NOTE: The old semicolon-style if syntax used in some existing action bodies must also
be updated during action body authoring (Plans 05/06). The grammar supports brace style only.
If LALR has conflicts with nested if, fall back to Earley (the fallback is already in place).

After all extensions, run:
  pytest tests/test_pycca.py -x -q   (all tests must pass)
  pytest tests/test_validation.py -x -q   (no regressions)

If LALR fails to compile with the extended grammar, the existing Earley fallback will
activate automatically. Document this in the SUMMARY if it occurs.
