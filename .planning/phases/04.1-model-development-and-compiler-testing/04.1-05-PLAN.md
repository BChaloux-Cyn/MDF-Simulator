<interfaces>
Pycca action body syntax (from extended grammar, Plan 04):
  Assignment:          self.attr = expr;
  Generate no params:  generate EventName to var;
  Generate with params: generate EventName(param: val) to var;
  Select from instances: select any var from instances of Class where expr;
  Select related by:   select any var related by self->RN;
  If/else:             if (expr) { stmts } else { stmts }
  Create:              create var of Class;
  Relate:              relate var to self across RN;
  Delete:              delete var;

Referential naming (COMPILATION.md): r{N}_{target_identifier_attr}
  R11: Elevator->Shaft (shaft_id)   -> r11_shaft_id  (on Elevator; Shaft has no referential)
  R12: Elevator->ElevatorIndicator  -> r12_indicator_id (on Elevator side)
  R10: Elevator->FloorIndicator     -> r10_elevator_id (on FloorIndicator)
  R13: Floor->FloorIndicator        -> r13_floor_num (on FloorIndicator)
  R1:  Elevator->Door               -> r1_elevator_id (on Door)

rcvd_evt prefix: event parameters are accessed as rcvd_evt.param_name
  ElevatorIndicator.Floor_updated has param floor_num -> rcvd_evt.floor_num
  Shaft.Floor_sensor_triggered has param floor_num -> rcvd_evt.floor_num
  FloorIndicator.Direction_set has param direction -> rcvd_evt.direction

Class diagram associations for traversal:
  Shaft related by self->R11 reaches Elevator (R11: Elevator 1 -- 1 Shaft)
  Elevator related by self->R12 reaches ElevatorIndicator
  FloorIndicator.r10_elevator_id is the referential to Elevator
  FloorIndicator.r13_floor_num is the referential to Floor

SHAFT.YAML — Fix the two Floor_sensor_triggered transition actions.

Both Moving_Up and Moving_Down have Floor_sensor_triggered transitions with the same
action body. The current action body has three errors (from RESEARCH.md Pitfall 4):
  - "self.current_floor = floor_num;" — bare floor_num, must be rcvd_evt.floor_num
  - "select any elev from instances of Elevator where R11;" — invalid where clause
  - "generate Floor_reached to elev;" — missing floor_num param
  - same errors for ElevatorIndicator select

Replace both (Moving_Up and Moving_Down) Floor_sensor_triggered action bodies with:
    action: >-
      self.current_floor = rcvd_evt.floor_num;
      select any elev related by self->R11;
      generate Floor_reached(floor_num: self.current_floor) to elev;
      select any ind related by elev->R12;
      generate Floor_updated(floor_num: self.current_floor) to ind;

Note: "select any ind related by elev->R12" traverses from the just-selected elev
variable. This requires the grammar to support non-self as the traversal source.
If the grammar only supports self->RN traversal, use this alternative:
    select any ind from instances of ElevatorIndicator where r12_elevator_id == elev.elevator_id;

Check what form was implemented in Plan 04 grammar extension (read 04.1-04-SUMMARY.md)
and use whichever form is supported.
