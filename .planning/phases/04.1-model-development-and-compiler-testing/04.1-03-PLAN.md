Three targeted changes to Elevator.yaml.

CHANGE 1 (ELV-006): Add Door_closed to the events list after Elevator_available:
  - name: Door_closed

CHANGE 2 (ELV-005 + Plan 02 reference updates): Replace the At_Floor state entry_action.
The current version uses old referential names and generates to class names incorrectly.
The corrected entry_action body (use YAML >- folded scalar):

      select any next_req from instances of Request where request_id == self.r14_request_id;
      if next_req != empty;
        if next_req.destination_floor > self.current_floor; self.next_direction = Up; end if;
        if next_req.destination_floor < self.current_floor; self.next_direction = Down; end if;
      else;
        select any fc from instances of FloorCall where floor_num == self.current_floor;
        if fc != empty; self.next_direction = fc.direction;
        else; self.next_direction = Stopped; end if;
      end if;
      select any floor_inst from instances of Floor where floor_num == self.current_floor;
      select any indicator from instances of FloorIndicator where r10_elevator_id == self.elevator_id and r13_floor_num == self.current_floor;
      generate Direction_set(direction: self.next_direction) to indicator;
      generate Elevator_arrived to floor_inst;
      generate Open_door to self;

Key corrections: r_queue_head_id -> r14_request_id; r10_floor_num -> r13_floor_num;
generates to bound variables not class names; Direction_set includes direction param.

CHANGE 3 (ELV-006): Replace the At_Floor->Idle transition.
Remove the existing single transition:
  from: At_Floor, to: Idle, event: Service_complete

Add TWO transitions:
First (Door_closed relay):
  - from: At_Floor
    to: Idle
    event: Door_closed
    action: >-
      select any req from instances of Request where r2_elevator_id == self.elevator_id;
      if req != empty; generate Door_closed to req; end if;

Second (fallback):
  - from: At_Floor
    to: Idle
    event: Service_complete

After changes, update Fix Applied sections in issues/elevator-005 and issues/elevator-006.
Door.yaml does NOT need changes — it already generates Door_closed to r1_elevator_id.

DISPATCHER.YAML changes (ELV-007):

The Dispatcher lacks a formal association to FloorCall. The ELV-007 issue recommends
Option A (explicit association) but that association has not yet been added to class-diagram.yaml
in Plan 02. For now, use select from instances of FloorCall as a pragmatic workaround.
The clean association will be added when the BoK documents the canonical queue idiom (Phase 4.2).

Replace all four transition action fields:

Transition: Idle -> Idle, guard: has_idle_elevator == True
Old action: "create object of Request;"
New action:
    action: >-
      select any fc from instances of FloorCall where floor_num != 0;
      create req of Request;
      req.destination_floor = fc.floor_num;
      select any elev from instances of Elevator;
      generate Request_assigned(target_floor: req.destination_floor) to elev;

Transition: Waiting -> Waiting, guard: queue_length > 1
Old action: "create object of Request; generate Request_assigned to Elevator;"
New action: (same as above)
    action: >-
      select any fc from instances of FloorCall where floor_num != 0;
      create req of Request;
      req.destination_floor = fc.floor_num;
      select any elev from instances of Elevator;
      generate Request_assigned(target_floor: req.destination_floor) to elev;

Transition: Waiting -> Idle, guard: queue_length == 1
Old action: "create object of Request; generate Request_assigned to Elevator;"

    action: >-
      select any fc from instances of FloorCall where floor_num != 0;
      create req of Request;
      req.destination_floor = fc.floor_num;
      select any elev from instances of Elevator;
      generate Request_assigned(target_floor: req.destination_floor) to elev;
