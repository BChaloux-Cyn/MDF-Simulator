## Decisions Made

- DestFloorButton and FloorCallButton must not re-declare `button_id` — it is inherited from `CallButton` via R6. The validator correctly flags this as a subtype identifier re-declaration error.
- FloorIndicator `Showing_Up` and `Showing_Down` states require `Direction == Stopped` transitions to `Off` — guard completeness validator checks all enum values for `Direction` (Up, Down, Stopped) on every state receiving a `Direction_set` event.
- Dispatcher `Idle` entry_action sets `self.has_idle_elevator = True` so the `has_idle_elevator` attribute is initialized correctly when elevator becomes available.
- `At_Floor` uses `select any indicator related by self->R10` (relationship traversal) rather than a where-clause with a referential — cleaner and consistent with the grammar's extended `select_related_stmt`.
