```python
class InstanceRegistry:
    def __init__(self, class_defs: dict[str, ClassManifest]):
        """Initialize with class definitions from domain manifest.
        
        class_defs maps class_name -> ClassManifest dict.
        """

        self._store: dict[str, dict[frozenset, dict]] = {}
```

Methods (all return lists of MicroStep, never raise exceptions per D-28):

- `create_sync(class_name: str, identifier: dict, initial_state: str, attrs: dict | None = None) -> list[MicroStep]`
  Per D-05: validate class exists and is not abstract (D-03). Build instance dict with identifier attrs + declared attrs (defaults from ClassManifest.attributes) + any overrides from `attrs` param. Per D-04: if class has a supertype, merge supertype's declared attributes into instance. Set `curr_state` to `initial_state`. Store in `_store` keyed by `make_instance_key(identifier)`. Return `[InstanceCreated(class_name=..., instance_id=identifier, initial_attrs=instance_dict, mode="sync")]`. If class is abstract, return `[ErrorMicroStep(error_kind="abstract_instantiation", message=f"Cannot instantiate abstract class {class_name}", context={"class": class_name})]`.

- `create_async(class_name: str, identifier: dict, initial_state: str, attrs: dict | None = None) -> tuple[list[MicroStep], Event | None]`
  Per D-05: same validation as sync. Build instance dict but do NOT set curr_state yet (creation event does that). Store the instance. Return `([InstanceCreated(...)], Event(event_type="__creation__", sender_class=class_name, sender_id=identifier, target_class=class_name, target_id=identifier, args={"initial_state": initial_state}))`.

- `delete_sync(class_name: str, identifier: dict) -> list[MicroStep]`
  Per D-06: lookup instance, remove immediately. Return `[InstanceDeleted(..., mode="sync")]`. If not found, return `[ErrorMicroStep(error_kind="double_deletion", ...)]`.

- `delete_async(class_name: str, identifier: dict) -> Event | None`
  Per D-06: validate instance exists (return None if not found, or ErrorMicroStep). Return `Event(event_type="__deletion__", ..., target_class=class_name, target_id=identifier)`. Instance is NOT removed yet — removal happens when deletion event is processed.

- `process_deletion(class_name: str, identifier: dict) -> list[MicroStep]`
  Called when deletion event is actually processed. Removes instance from store. Returns `[InstanceDeleted(..., mode="async")]`.

- `lookup(class_name: str, identifier: dict) -> dict | None`
  Return instance dict or None.

- `lookup_all(class_name: str) -> list[dict]`
  Return all instances of the given class.

- `get_state(class_name: str, identifier: dict) -> str | None`
  Return `curr_state` of the instance, or None if not found.

- `set_state(class_name: str, identifier: dict, new_state: str) -> None`
  Update `curr_state`. Called by scheduler during transition.

- `set_attr(class_name: str, identifier: dict, attr_name: str, value: Any) -> None`
  Update an attribute on the instance dict.

- `get_attr(class_name: str, identifier: dict, attr_name: str) -> Any`
  Read an attribute from the instance dict.

Create `engine/relationship.py` with class `RelationshipStore`:

```python
class RelationshipStore:
    def __init__(self, associations: dict[str, AssociationManifest]):
        """Initialize with association definitions from domain manifest.
        
        associations maps rel_id -> AssociationManifest dict.
        """
        self._associations = associations
        # {rel_id: set of (frozenset_key_a, class_a, frozenset_key_b, class_b)}
        self._links: dict[str, set[tuple[frozenset, str, frozenset, str]]] = {
            rel_id: set() for rel_id in associations
        }
```

Methods (return lists of MicroStep, never raise per D-28):

- `relate(rel_id: str, class_a: str, id_a: dict, class_b: str, id_b: dict) -> list[MicroStep]`
  Per D-07: validate rel_id exists. Per D-08: check multiplicity before adding. For "1" multiplicity on either side, count existing links involving that instance on that side — if already at 1, return `[ErrorMicroStep(error_kind="multiplicity_violation", message=f"Multiplicity violation on {rel_id}: {class_a} already linked", context={"rel_id": rel_id, "class": class_a, "id": id_a})]`. If valid, add link tuple to `_links[rel_id]`. Return empty list (no micro-step for successful relate — it's a side effect within an action).

  Multiplicity interpretation from AssociationManifest:
  - `mult_a_to_b`: "1" means each A links to exactly one B; "M" or "*" means many
  - `mult_b_to_a`: "1" means each B links to exactly one A; "M" or "*" means many
  - Check direction: when adding a link, if `mult_b_to_a == "1"`, verify instance B has no existing link on this rel. If `mult_a_to_b == "1"`, verify instance A has no existing link on this rel.

- `unrelate(rel_id: str, class_a: str, id_a: dict, class_b: str, id_b: dict) -> list[MicroStep]`
  Find and remove the matching link. If not found, return `[ErrorMicroStep(error_kind="unrelate_not_found", ...)]`. Return empty list on success.

- `navigate(rel_id: str, from_class: str, from_id: dict) -> list[dict]`
  Per D-09: find all linked instances on the OTHER side of rel_id from the given instance. Return a list of `{"class": class_name, "id": identifier_dict}` dicts. If the "to" multiplicity is "1", the list has 0 or 1 elements. If "M", it may have 0..N elements.

- `navigate_chain(chain: list[tuple[str, str]], start_class: str, start_id: dict) -> list[dict]`
  Per D-09: chained traversal like `self->R1->R2->R3`. The `chain` is a list of `(rel_id, target_class)` tuples. Navigate each hop, collecting results. Return final targets.
