# Phase 05.1 Plan 01: Engine Data Types and Test Scaffold Summary

Defined the shared runtime data types (12 micro-steps + ErrorMicroStep, Event, DomainManifest TypedDicts, instance-key helper) and Wave 0 test scaffold so every subsequent plan in Phase 05.1 imports stable contracts instead of inventing them ad hoc.

## What Shipped

- `engine/microstep.py` — base `MicroStep` plus 12 frozen dataclass variants (`SchedulerSelected`, `EventReceived`, `GuardEvaluated`, `TransitionFired`, `ActionExecuted`, `GenerateDispatched`, `EventDelayed`, `EventDelayExpired`, `EventCancelled`, `InstanceCreated`, `InstanceDeleted`, `BridgeCalled`) and `ErrorMicroStep` per D-28. Each carries a `Literal` `type` field assigned via `field(default=..., init=False)`.
- `engine/event.py` — `Event` dataclass (event_type, sender_class, sender_id, target_class, target_id, args, delay_ms) and `make_instance_key` helper that returns `frozenset(sorted(items))` for order-independent hashing.
- `engine/manifest.py` — `TransitionEntry`, `ClassManifest`, `AssociationManifest`, `DomainManifest` TypedDicts with no runtime dependencies.
- `engine/__init__.py` — re-exports the public surface.
- `tests/test_engine.py` — 3 implemented tests (`test_microstep_types`, `test_event_and_instance_key`, `test_engine_isolation_no_schema_imports`) and 36 skipped stubs cross-referencing the plans that will implement them. Includes a hand-written `DOMAIN_MANIFEST` fixture for TrafficLight + Controller and a `BRIDGE_MOCKS` constant.
- `tests/fixtures/bridge_mocks.yaml` — 4 mock operations.
