# Phase 05.1 Plan 02: Instance Registry and Relationship Store Summary

Built the two stateful runtime stores — `InstanceRegistry` (instance lifecycle, lookup, attr/state access) and `RelationshipStore` (link create/remove, multiplicity-checked navigation) — that every higher-level Phase 05.1 component (scheduler, ctx, dispatcher) will sit on top of.

## What Shipped

- `engine/registry.py` — `InstanceRegistry`
  - `create_sync(class_name, identifier, initial_state, attrs=None)` returns `[InstanceCreated(mode="sync")]`; rejects abstract classes with `ErrorMicroStep(error_kind="abstract_instantiation")`
  - `create_async(...)` returns `([InstanceCreated(mode="async")], Event(event_type="__creation__"))`; instance is stored immediately with `curr_state=None`
  - `delete_sync` removes immediately; `delete_async` returns an `Event(event_type="__deletion__")` and defers removal until `process_deletion`
  - Double-deletion surfaces as `ErrorMicroStep(error_kind="double_deletion")` (D-28)
  - Supertype attribute inheritance per D-04 — walks the `supertype` chain merging declared attribute defaults
  - `lookup`, `lookup_all`, `get_state`/`set_state`, `get_attr`/`set_attr` accessors
- `engine/relationship.py` — `RelationshipStore`
  - `relate(rel_id, class_a, id_a, class_b, id_b)` enforces multiplicity by counting existing links on the "1" side; violations return `ErrorMicroStep(error_kind="multiplicity_violation")`
  - `unrelate` returns `ErrorMicroStep(error_kind="unrelate_not_found")` if no matching link
  - `navigate(rel_id, from_class, from_id)` works from either side of the association and returns `[{"class", "id"}, ...]`
  - `navigate_chain(chain, start_class, start_id)` walks `[(rel_id, target_class), ...]` hops, deduping per hop
- `tests/test_engine.py` — 11 new tests (8 registry, 3 relationship), all stubs unskipped, plus a `Vehicle/Car` mini-manifest for the supertype-inheritance test and an `A/B/C` mini-manifest for chained navigation
