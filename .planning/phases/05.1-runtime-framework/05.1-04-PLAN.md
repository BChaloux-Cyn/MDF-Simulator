Create `engine/scheduler.py` with class `ThreeQueueScheduler`:

```python
import heapq
from collections import deque
from typing import Any, Callable, Generator

from engine.event import Event, make_instance_key
from engine.microstep import (
    MicroStep, SchedulerSelected, EventReceived, GuardEvaluated,
    TransitionFired, ActionExecuted, GenerateDispatched, EventDelayed,
    EventDelayExpired, EventCancelled, ErrorMicroStep,
)
from engine.registry import InstanceRegistry
from engine.clock import SimulationClock
```

Constructor takes: `registry: InstanceRegistry`, `clock: SimulationClock`, `class_defs: dict[str, ClassManifest]`, `generalizations: dict[str, list[str]]` (maps supertype -> list of subtypes).

Internal state:
- `self._priority: deque[Event]` — FIFO, self-directed events (D-10)
- `self._standard: deque[Event]` — FIFO, cross-instance + creation + deletion events (D-11)
- `self._delay: list[tuple[float, int, Event]]` — min-heap by (expiry_time, sequence_number, event). Sequence number is a monotonic counter for deterministic tie-breaking per pitfall 2.
- `self._seq: int` — monotonic counter for delay queue tie-breaking
- `self._pending_generated: list[Event]` — run-to-completion buffer (D-18, D-19)
- `self._in_flight: Event | None` — the event currently being processed

Key methods:

**`enqueue(event: Event) -> list[MicroStep]`**
Determine which queue per rules 1-5:
- If `event.delay_ms is not None and event.delay_ms > 0`: delay queue. Per D-14, check for existing delayed event with same (event_type, sender, target) — if found, remove it first. Add `(clock.now() + event.delay_ms, self._seq, event)` to heap. Increment `_seq`. Return `[EventDelayed(...)]`.
- If `event.sender_id == event.target_id and event.sender_class == event.target_class`: priority queue. But if `self._in_flight is not None` (inside run-to-completion), add to `_pending_generated` instead. Return `[GenerateDispatched(..., queue="priority")]`.
- Else: standard queue. Same pending buffer logic if in-flight. Return `[GenerateDispatched(..., queue="standard")]`.
- Creation events (`event_type == "__creation__"`): always standard queue (rule 3).

**`cancel(event_type: str, sender_id: dict, target_id: dict) -> list[MicroStep]`**
Per D-25: scan delay queue for matching (event_type, sender_key, target_key). Remove if found, re-heapify. Return `[EventCancelled(...)]` or empty list.

**`tick_delay_queue() -> list[MicroStep]`**
Per rule 8: pop all entries where `expiry_time <= clock.now()`, append events to standard queue. Return `[EventDelayExpired(...)]` for each.

**`execute() -> Generator[MicroStep, None, None]`**
The main scheduler loop per rules 6-9:
```
while True:
    yield from self.tick_delay_queue()
    if self._priority:
        event = self._priority.popleft()
        yield SchedulerSelected(queue="priority", ...)
        yield from self._process_event(event, queue="priority")
    elif self._standard:
        event = self._standard.popleft()
        yield SchedulerSelected(queue="standard", ...)
        yield from self._process_event(event, queue="standard")
    else:
        break  # all queues empty — domain idle (rule 9)
```

**`_process_event(event: Event, queue: str) -> Generator[MicroStep, None, None]`**
1. Resolve target: look up instance in registry. If `event.target_class` is a supertype (per generalizations), resolve to concrete subtype class via registry lookup. Per D-20, polymorphic dispatch. If instance not found, yield `ErrorMicroStep(error_kind="unknown_target", ...)` and return.
2. Handle special events: `__creation__` (set curr_state from args["initial_state"], run entry action if any), `__deletion__` (run final action, then `registry.process_deletion()`).
3. For normal events: yield `EventReceived(...)`. Look up `(curr_state, event_type)` in transition table of the resolved class.
4. If not in table: yield `ErrorMicroStep(error_kind="cant_happen", ...)` per D-21.
5. If entry is `None` for next_state (event ignored): do nothing more per D-21.
6. If guard_fn exists: call it, yield `GuardEvaluated(...)`. If false, return (no transition).
7. Execute transition: set `self._in_flight = event`. If action_fn exists, call it (action_fn receives instance dict and event args). Yield `ActionExecuted(...)`. The action_fn may call back into the scheduler's `enqueue()` via a callback passed as parameter. Those generated events go to `_pending_generated`.
8. Update state: `registry.set_state(target_class, target_id, next_state)`. Yield `TransitionFired(from_state=old_state, to_state=next_state, ...)`.
9. Final state check (D-23, D-24): if `next_state` is in `class_defs[target_class]["final_states"]`, post async deletion event via `registry.delete_async()` and enqueue the returned Event.
10. Flush pending: set `self._in_flight = None`. For each event in `_pending_generated`, call `self._enqueue_direct()` (bypasses buffer since we're no longer in-flight). Clear buffer.

IMPORTANT — the `enqueue` method called by action_fn callbacks must detect `self._in_flight is not None` and buffer to `_pending_generated` instead of directly enqueueing. This is the core run-to-completion mechanism per D-18/D-19.

IMPORTANT — Transition table convention: `TransitionEntry` with `next_state=None, action_fn=None, guard_fn=None` means "event ignored". Absence from the table entirely means "can't happen".
