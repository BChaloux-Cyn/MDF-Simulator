## Deviations from Plan

**1. [Rule 1 - Bug] test_all_microstep_types initially missed bridge_called**
- **Found during:** Task 2 first run
- **Issue:** `ctx.bridge()` invoked from inside an action_fn does emit a BridgeCalled, but the action's return value is dropped by the scheduler — the micro-step never reached the test's collected list
- **Fix:** Added a top-level `ctx.bridge("LogEvent", {"phase": "init"})` call before the scenario, capturing its returned step into the steps list directly
- **Files modified:** `tests/test_engine.py`
- **Commit:** rolled into `4b00f81`

**2. [Rule 1 - Bug] test_integration_action_callback_wiring asserted GenerateDispatched in execute() stream**
- **Found during:** Task 3 first run
- **Issue:** The scheduler buffers events generated mid-action into `_pending_generated` and returns `GenerateDispatched` from `enqueue()`, but `execute()` never re-yields these returns — they only exist as the action callback's return value
- **Fix:** Test now captures `gen_steps = ctx.generate(...)` inside the action and asserts the captured list, not the execute() stream
- **Secondary issue:** Action originally passed the full instance dict (which contains `curr_state`) as the target_id, producing a frozenset key mismatch -> `unknown_target` ErrorMicroStep
- **Fix:** Action now constructs `light_key = {"light_id": instance["light_id"]}` for sender/target ids
- **Files modified:** `tests/test_engine.py`
- **Commit:** rolled into `4667ca1`

No architectural deviations. No Rule 4 questions.
