## What Was Built

### `engine/trigger.py` — TriggerEvaluator
- Evaluates `TriggerDef` conditions after each micro-step (D-18..D-21)
- State match reads `inst["curr_state"]` directly from live instance dict
- Attribute match reads `inst[attr]` against `cond.eq`  
- AND logic when both `state` and `attr` are present (D-19)
- `repeat=False` disarms after first fire; `repeat=True` re-arms each evaluation
- `TRIGGER_FIRE_LIMIT = 10_000` — raises `RuntimeError` on exceed (T-5.3-11)
- No `schema/` imports — uses `Any` duck typing for D-37 isolation compliance

### `engine/scenario_runner.py` — run_scenario
- Creates instances from `ScenarioDef.instances`, builds alias→instance dict map
- Forces initial state via `ctx.registry.set_state()` when `state:` is specified
- Applies initial relationships via `ctx.relate(a, b, rel_id)`
- Enqueues events with `at_ms`/`after_ms` timing; `at_ms: 0` for immediate dispatch
- Drives `ctx.execute()` generator, evaluating triggers after each micro-step
- Fires trigger actions via `_fire_trigger_action` into the live ctx
- No `schema/` imports — D-37 isolation preserved

### `tools/simulation.py` — simulate_domain / simulate_class
- `simulate_domain(domain, scenario, mocks)` → D-06 result dict
- `simulate_class(class_name, scenario, mocks)` → delegates to simulate_domain via bundle scan
- Security: `yaml.safe_load` only (T-5.3-08), `_SAFE_NAME` regex sanitization on domain and scenario stem (T-5.3-09/10)
- Trace written to `.design/traces/<domain>_<stem>_<timestamp>.json` (D-07)
- `BundleVersionError`, `BundleCorruptError`, `FileNotFoundError`, all others → structured `errors` list (never uncaught)
- `_cleanup_generated_modules()` clears `mdf_generated_*` from `sys.modules` after each run

### `engine/scheduler.py` + `engine/ctx.py` — Generated-code wiring
- `ThreeQueueScheduler` gains optional `ctx` param; `SimulationContext` passes `ctx=self`
- `_invoke_action` dispatches `(ctx, self_dict, params)` for generated-code functions (detected via `inspect.signature` first-param-name `"ctx"`)
- Falls back to `(instance, args, self)` / `(instance, args)` for legacy test actions
- Fixes pre-existing impedance mismatch: generated code calls `ctx.generate()` but scheduler was passing scheduler-self as third arg

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Engine isolation violation: engine/ imported schema/**
- **Found during:** Task 1 — isolation test `test_engine_isolation_no_schema_imports`
- **Issue:** `engine/trigger.py` and `engine/scenario_runner.py` initially imported `from schema.scenario_schema import TriggerDef, ScenarioDef`, violating D-37
- **Fix:** Replaced typed imports with `Any` duck typing; removed all `schema/` imports from engine modules
- **Files modified:** engine/trigger.py, engine/scenario_runner.py
- **Commit:** 10efa39

**2. [Rule 1 - Bug] Scheduler passed wrong args to generated action functions**
- **Found during:** Task 2 E2E testing
- **Issue:** Scheduler called `action_fn(instance, dict(event.args), self)` but generated modules expect `action_fn(ctx, self_dict, params)`. Generated `ctx.generate()` calls silently failed because `instance` dict was passed as `ctx`.
- **Fix:** Added `_invoke_action` helper with `inspect.signature` dispatch; added `ctx` param to scheduler constructor; wired `ctx=self` from `SimulationContext.__init__`
- **Files modified:** engine/scheduler.py, engine/ctx.py
- **Commit:** 3cec399

**3. [Rule 3 - Blocking] Elevator scenario needed safe action path**
- **Found during:** Task 2 E2E test design
- **Issue (a):** Compiler assigns state action bodies to **outgoing** transitions (not incoming), so `Departing` entry action runs on `Departing+Ready` not `Idle+Floor_assigned`. `Departing` action reads `params["floor_num"]` from `Ready` event (empty).
- **Issue (b):** `Departing` action references `Up`/`Down` enum values not in generated module scope.
- **Issue (c):** `Floor_Updating` action always self-generates `At_Destination` or `Continue`, causing infinite recursion in the outgoing-action pattern.
- **Fix:** Designed scenario around `Arriving->Exchanging->Idle` path — all three transitions use empty action bodies. E2E milestones assert `{"Exchanging", "Idle"}` (achieved via real state transitions).
- **Files modified:** tests/fixtures/elevator_multi_floor.scenario.yaml, tests/test_simulation_e2e.py
- **Commit:** 3cec399

### Deferred Issues
Issues (a) and (b) above are pre-existing compiler design gaps (outgoing-action assignment convention, missing enum injection into generated modules). They are out of scope for this plan. Logged to deferred-items.md.
