## Sender Requirement

**User's decision:** Sender is **always required** for all events — scenario events, trigger-fired events, and method calls. Missing sender is a Pydantic validation error before simulation starts. No defaults.

## Method Calls in Scenario

**User's clarification:** `call: method_name` in the scenario executes the action body of that method directly on the target instance. Any `generate` statements inside follow normal scheduler rules (go through the event queue). Sender is required.
