# Phase 05.3 Plan 02: ctx API Gap — Instance Key + Generated-Code API Summary

**One-liner:** Instance dicts now carry `__instance_key__` (frozenset) and `__class_name__` (str), and `SimulationContext` gains the full generated-code-facing API (`create`, `delete`, `relate`, `traverse`, `select_any_related`, etc.) with type-dispatch overloading that preserves all existing scenario-loading callers.

## What Was Built

### Task 1 — registry.py

`_build_instance` now accepts a `class_name` parameter and sets two metadata slots on every instance dict before returning:

```python
instance["__instance_key__"] = make_instance_key(identifier)
instance["__class_name__"] = class_name or cdef.get("name", "")
```

Both `create_sync` and `create_async` pass `class_name` through. Composite identifiers (multiple attrs) produce a frozenset of all `(name, value)` pairs.

### Task 2 — ctx.py

New methods on `SimulationContext`:

| Method | Shape | Notes |
|--------|-------|-------|
| `create(class, attrs)` | Generated-code | Returns instance dict; splits identifier from attrs via ClassManifest |
| `delete(inst_dict)` | Generated-code | Overloads existing `delete(class, id)` via isinstance dispatch |
| `delete_where(class, pred)` | Generated-code | Catches predicate exceptions silently |
| `relate(a, b, rel_id)` | Generated-code | Overloads existing `relate(rel_id, class_a, id_a, class_b, id_b)` |
| `unrelate(a, b, rel_id)` | Generated-code | Same dispatch |
| `traverse(source, rel_chain)` | Generated-code | Walks relationship chain, returns full instance dicts from registry |
| `select_any_related(src, chain)` | Generated-code | `traverse()[0]` or None |
| `select_many_related(src, chain)` | Generated-code | `traverse()` |
| `generate(..., target=frozenset, sender=dict)` | Generated-code | Detects frozenset target, resolves class from registry scan |
| `cancel(..., sender=dict, target=dict)` | Generated-code | Overloads existing cancel() |
| `bridge(domain, op, args)` | Generated-code | Overloads existing `bridge(op, args)` returning `(value, steps)` |

Helper `_resolve_class_for_key(frozenset)` scans `registry._store` to find the owning class for a given instance key (needed by `generate` frozenset path).
