Guard completeness logic (from CONTEXT.md + RESEARCH.md):
```
1. Group transitions by (from_state, event)
2. For each group:
   a. Multiple unguarded transitions → error (ambiguous)
   b. No guard expressions → skip (unguarded is OK if only one)
   c. Parse each guard with GUARD_PARSER
   d. If parse fails → warning (cannot analyze)
   e. Extract (variable, operator, value) from simple_compare trees
   f. Determine variable type from types_map
   g. If type is String → error (strings forbidden in guards)
   h. If type is Enum → check all enum values appear in guard set → error if gap
   i. If type is Integer/Real with range → interval coverage check → error if gap
   j. If type is Integer/Real without range → inequality interval analysis → warning if gap
   k. If AND/OR expression detected → cannot determine completeness → warning
```

Interval normalization (from RESEARCH.md):
```
"<"  N → (-∞, N)     (half-open, excludes N)
"<=" N → (-∞, N+1)   (half-open, treat as < N+1 for integer; for Real: < N+ε)
">"  N → (N+1, +∞)   (half-open)
">=" N → (N, +∞)     (half-open)
Gap detection: sort intervals, check adjacent pairs for gaps
```

<task type="auto">
  <name>Task 2: Implement _check_guard_completeness in tools/validation.py</name>
  <files>tools/validation.py</files>
  <action>
    Add `_check_guard_completeness(sd: StateDiagramFile, domain: str, types_map: dict | None) -> list[dict]` to `tools/validation.py` and call it from the per-active-class validation pipeline (in `_validate_domain_data`, after `_check_referential_integrity_state_diagram`).

    **Implementation steps:**

    1. Import GUARD_PARSER at module level:
       ```python
       from pycca.grammar import GUARD_PARSER
       ```

    2. Group transitions by (from_state, event) using `itertools.groupby` (same pattern as yaml_schema.py guard consistency check):
       ```python
       from itertools import groupby
       key = lambda t: (t.from_state, t.event)
       for (from_state, event), group in groupby(sorted(transitions, key=key), key=key):
           ts = list(group)
       ```

    3. **Multiple unguarded check:** if len(ts) > 1 and all guards are None → error (ambiguous).

    4. **Skip if no guards:** if all guards are None and len(ts) == 1 → nothing to check.

    5. **Parse each guard expression** using GUARD_PARSER.parse(t.guard). Wrap in try/except — parse failure → warning, skip this group.

    6. **Detect AND/OR expressions** (tree.data in {"and_expr", "or_expr"} at top level) → warning, skip interval analysis for this group.

    7. **Extract variable and type** from `simple_compare` tree:
       - variable = str(tree.children[0])  (Token NAME)
       - Determine type: look up variable in class attributes via class-diagram... but in _check_guard_completeness we have sd (state diagram) not class diagram. The guard variable is typically an event parameter or a class attribute name. Look up in sd.events[*].params first; if not found, fall back to types_map keys. If type still unknown → skip (emit no issue — cannot determine type without full cross-reference).
       - Actually: guard expressions in transitions reference EVENT PARAMETERS. Look up the guard variable name in the event's params (find the matching event in sd.events where event.name == transition.event). If not found in event params, skip completeness analysis.

    8. **String type check:** if param.type == "String" (or a type in types_map with base = a String-like type) → error. The five primitives: String is explicitly forbidden in guards per CONTEXT.md.

    9. **Enum type check:** look up param.type in types_map. If it's an EnumType → collect all guard expressions for this (from, event) group → extract the compared value from each guard → find enum values not covered → error if any missing.

    10. **Integer/Real interval analysis:**
        - Normalize each guard to a half-open interval using the RESEARCH.md mapping.
        - Sort intervals by lower bound.
        - Check adjacent pairs for gaps.
        - If range is defined in ScalarType → check coverage against [min, max].
        - Report gaps as `severity="warning"` (per CONTEXT.md: "Integer or Real with no range defined → inequality interval analysis" is a warning, not error).
        - Exception: if range IS defined in types.yaml, gaps are errors not warnings.

    **Integration:** In `_validate_domain_data`, after loading and validating each active class's state diagram, add:
    ```python
    types_map = _load_types_map(domain_path)  # loads types.yaml if present, returns dict[str, TypeDef] or None
    issues += _check_guard_completeness(sd, domain, types_map)
    ```

    **Helper `_load_types_map(domain_path: Path) -> dict | None`:** loads types.yaml, validates with TypesFile, returns {type.name: type_def} dict. Returns None if file absent. Returns None if parse fails (issues from that failure go through standard validation, not guard completeness).
