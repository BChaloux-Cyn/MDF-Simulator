  <name>Task 2: Implement tools/validation.py and register MCP tools in server.py</name>
  <files>tools/validation.py, server.py</files>
  <action>
    Implement `tools/validation.py` with three public tool functions and private helpers. Structure:

    **Module-level constants:**
    ```python
    from pathlib import Path
    import yaml
    import networkx as nx
    from pydantic import ValidationError
    from schema.yaml_schema import (
        ClassDiagramFile, StateDiagramFile, DomainsFile, TypesFile,
        _SCALAR_PRIMITIVES,
    )
    from tools.model_io import _resolve_domain_path, _pydantic_errors_to_issues

    MODEL_ROOT = Path(".design/model")
    ```

    **`_make_issue(issue, location, value=None, fix=None, severity="error") -> dict`**
    Returns the extended issue dict format with all five fields.

    **`_load_yaml_file(path: Path) -> tuple[dict | None, list[dict]]`**
    Loads and YAML-parses a file. Returns (data, []) on success, (None, [issue]) on YAML error. Never raises.

    **`_validate_domain_data(domain: str, domain_path: Path, report_missing: bool) -> list[dict]`**
    Core per-domain validation. Calls:
    - `_check_missing_class_diagram(domain, domain_path, report_missing)`
    - Loads class-diagram.yaml, validates with `ClassDiagramFile.model_validate()`
    - Collects class names, association R-numbers, event names per class-diagram
    - `_check_referential_integrity_class_diagram(cd, domain)` — checks:
      - association.point_1, association.point_2 in class_names
      - subtype.specializes R-number in associations
      - associative.formalizes R-number in associations
      - attribute.type is primitive or in types (loads types.yaml if present)
      - method.return_type and method param types
      - bridge operations vs DOMAINS.yaml (only within validate_model scope — skip in validate_domain/validate_class since DOMAINS.yaml is not loaded)
    - For each `active` class:
      - Check if state diagram file exists (with subtype inheritance exception)
      - If missing and report_missing=True, append issue
      - If present, load and validate with `StateDiagramFile.model_validate()`
      - `_check_referential_integrity_state_diagram(sd, domain)`:
        - initial_state in {s.name for s in states}
        - transition.to in state names
        - transition.event in event names
      - `_check_reachability(sd, domain)`: build nx.DiGraph, run after referential integrity (skip if initial_state not found)

    **`_check_reachability(sd: StateDiagramFile, domain: str) -> list[dict]`**
    Build DiGraph from states/transitions. Run `nx.descendants(G, sd.initial_state)`. Report unreachable states as errors, trap states (G.out_degree == 0) as warnings. Guard: if initial_state not in G.nodes, return [] (referential integrity catches it first).

    **Public tool functions:**

    `validate_class(domain: str, class_name: str, report_missing: bool = True) -> list[dict]`:
    - Load class-diagram.yaml for domain (use _resolve_domain_path)
    - Find ClassDef with name == class_name
    - If not found, return [issue: class not found]
    - Run referential integrity on that class only
    - If class.stereotype == "active": load state diagram, run referential integrity + reachability
    - Never raises

    `validate_domain(domain: str, report_missing: bool = True) -> list[dict]`:
    - Call `_validate_domain_data(domain, domain_path, report_missing)`
    - Never raises

    `validate_model(report_missing: bool = True) -> list[dict]`:
    - Load DOMAINS.yaml from MODEL_ROOT. If absent and report_missing=True, return missing-file issue.
    - Validate DomainsFile schema.
    - For each domain in DomainsFile.domains: call `_validate_domain_data(domain.name, ...)`
    - Also check required/provided bridge operations against DOMAINS.yaml bridge declarations:
      - For each required_bridge in a class-diagram, each operation in required_bridge.operations must
        appear in the corresponding BridgeEntry.operations in DOMAINS.yaml. Report any undeclared operation.
      - For each provided_bridge, each BridgeImplementation.name must match a declared operation, and
        every declared operation must have an implementation. Report missing implementations.
    - Never raises

    **Anti-patterns to avoid (per RESEARCH.md):**
    - Never raise in any public function — wrap all file I/O and validation in try/except
    - Never return early on first error — accumulate all issues
    - Never call nx.descendants() before checking initial_state is in graph nodes
    - report_missing flag suppresses missing-file issue creation, not file reads — still run checks on files that exist
