<decisions_made>
- **MDFType structured type** (not raw strings) — enables type parameter extraction for
  argument checking (`put(k,v)` validates k matches Map's K param) and return type
  resolution (`Map<String,Integer>.get()` → `Optional<Integer>` via `param(1)`)
- **SymbolScope in compiler/symbol_table.py** — independently testable, clean seam for
  Phase C schema injection via DomainContext
- **TypedExpr wrapper** (not parallel type stack) — stack discipline is fragile; wrapper
  is self-documenting and testable; `__str__` transparency preserves statement method compat
- **Hard errors** (TypeMismatch / UnresolvedType) — user specified; well-formed MDF
  models have all types defined
- **transform_action() kwargs** for Phase C context — None defaults, full backward compat
- **All phases ship together** — user will implement A→E sequentially, single release
- **Struct field checking in Phase C** — DomainContext.attribute_type() + dotted_name
  already has both pieces; no new mechanism
- **Argument type checking in Phase E** — needs TypedExpr on arguments to know types;
  put(k,v)/remove(k)/get(k)/contains_key(k) all validated against receiver type params
</decisions_made>
